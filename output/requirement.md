### 3.3 需求背景与目标

鲨域现有课程系统已支持课程、视频、看课、答题、奖励等能力，但缺少独立的专栏式内容学习模块。本次新增【专栏课】，支持租户按“专栏 -> 课程 -> 章节”维护内容，并在客户端提供连续学习、章节目录、完播答题、完播奖励、装修入口和分享直达能力。章节播放设置、完播答题和完播奖励需要支持章节级配置，同时提供租户级默认配置降低重复配置成本。客户端章节视频播放与客户操作必须遵循管理后台【播放设置】，并与鲨域课程系统的视频播放完全一致。

### 3.4 管理后台

- 运营 -> 内容运营新增【专栏课】，采用左侧专栏列表、右侧课程列表结构。
- 专栏支持新增、编辑、删除、搜索、选择。
- 课程支持新增、编辑、删除、上下架、搜索、推广分享，不提供复制。
- 章节支持新增、编辑、复制、删除、上下架、排序、推广分享。
- 新增/编辑课程使用独立页面，所属专栏、课程名称、课程简介、主讲人、课程封面、课程图文详情、上下架状态等组件单列纵向展示，图文详情使用富文本编辑器。
- 课程详情页展示课程基本信息和章节列表，不展示管理端【课程介绍】。
- 新增/编辑章节页包含【基本信息】【播放设置】【运营设置】，所属课程固定展示且不可修改，章节图文详情使用富文本编辑器。
- 章节播放设置支持默认设置和自定义设置。自定义设置维护进度条、手动暂停、快进、已完播快进、继续播放、随机暂停和完播判定进度。
- 内容设置新增独立的【播放设置】和【默认奖励设置】页面，分别维护租户级专栏课默认播放策略和专栏课完播奖励。
- 章节运营设置包含完播答题和完播奖励。答题支持多个题目；奖励由开关控制，可选择默认奖励或自定义奖励，每章只能配置一种奖励，类型为积分、优惠券、现金红包单选。

### 3.5 客户端

- 装修组件新增专栏课选项卡，样式与课程选项卡一致，展示课程名称和课程介绍，点击进入专栏课详情页。
- 新增专栏课详情页、章节详情页和章节目录弹窗，支持装修入口、推广分享链接和小程序分享转发进入。
- 章节视频播放按最终生效的播放设置执行；默认设置取租户级策略，自定义设置取章节策略。
- 播放控制、进度条、手动暂停、快进、已完播快进、继续播放、随机暂停和完播判定与鲨域课程系统一致。
- 未登录用户可查看专栏课详情页和章节详情页并播放视频，学习进度保存在本地；登录后按更靠后的进度和更高完成状态与服务端合并。
- 用户完播后，开启答题时需全部题目答对才可领取奖励；未开启答题时可直接领取奖励。答错可继续答题直到答对。
- 未登录用户的提交答案按钮置灰且不可点击。
- 专栏课详情页标题为“课程”，章节列表标题为“章节目录”，章节详情页标题为“章节”。
- 课程和章节图文详情支持五段式图文混排，超过半屏时默认折叠并展示【展开详情】。
- 下架或删除的课程/章节不可作为正常可访问内容展示。

### 3.6 范围与非目标

- 本次包含管理后台专栏课、课程、章节管理，统一播放设置与默认奖励设置，客户端详情页、章节页、目录弹窗、装修组件、分享直达、未登录进度同步、完播答题和完播奖励闭环。
- 不将现有课程系统直接改造成专栏课系统，不把专栏课等同于现有训练营、课程或短视频内容。
- 不在产品阶段定稿服务归属、表结构、接口字段或文件级实现方案。
- 不把红包、积分、优惠券等底层发放能力定义为专栏课模块自有能力。
- 不单独拆分 APP 端产品页面，小程序和 APP 使用共用跨平台多端研发框架的一套客户端体验表达。

### 3.7 风险与依赖

- 完播奖励依赖积分、红包、优惠券底层发放能力和领取记录。
- 装修组件依赖专栏课数据源、装修配置保存和客户端渲染。
- 腾讯 VOD 上传/播放复用现有能力，需处理专栏课章节绑定和权限校验。
- 多题答题直到答对需避免重复领奖或绕过完播条件。
- 客户端播放设置需要与鲨域课程系统既有播放器配置、状态流转和完播判定规则对齐。

## 十、关键业务流程与测试场景（每步标注操作角色）

### 10.1 主流程 (Happy Path)

1. 租户运营人员：进入专栏课模块并选择专栏 -> 管理后台：展示该专栏下课程列表。（依据：验收标准第1条）
2. 租户运营人员：新增课程并维护基本信息 -> 管理后台：保存课程并在独立详情页展示基本信息和章节列表。（依据：需求内容第3、4条）
3. 租户运营人员：新增章节并选择默认或自定义播放策略 -> 管理后台：保存章节及其最终生效播放设置。（依据：需求内容第5、6、12条）
4. 租户运营人员：开启答题和奖励并完成配置 -> 管理后台：章节列表展示题目数量和单一实际奖励。（依据：需求内容第8、9条；验收标准第5条）
5. 客户端用户：从装修入口或分享链接进入详情并播放章节 -> 客户端：按最终生效播放设置控制视频并记录进度。（依据：需求内容第10至14条）
6. 登录用户：完成播放并答对全部题目 -> 客户端：允许领取该章节配置的奖励。（依据：需求内容第15条）

### 10.2 异常流程 (Alternative Paths)

- 默认播放设置未配置：章节选择默认设置时，系统给出配置缺失提示或引导配置。（依据：验收标准第7条）
- 未登录答题：未登录用户可播放视频，但提交答案按钮置灰且不可点击。（依据：验收标准第12条）
- 答题未全部正确：用户暂不可领取奖励，可继续答题直到全部答对。（依据：验收标准第11条）
- 资源已下架或删除：课程/章节不可作为正常可访问内容展示。（依据：验收标准第16条）
- 未授权：设置项禁用或操作被拦截，不允许越权修改；跨租户资源按不可访问处理。（依据：已审查原型 `special-column-tab.html`；用户确认，2026-07-18）
- 奖励请求超时：领取状态进入“发放中”，通过外部流水号查询最终结果；重复请求返回原领取结果。（依据：用户确认，2026-07-18）
- 匿名进度同步失败：保留本地进度并允许重试，不使用失败结果覆盖服务端更高状态。（依据：用户确认，2026-07-18）

### 10.3 边界与特殊场景 (Edge Cases)

- 图文详情超过半屏时默认折叠并显示【展开详情】。（依据：验收标准第14条）
- 课程不支持复制，章节支持复制。（依据：需求内容第2条）
- 每个章节只能配置一种奖励，积分、优惠券、现金红包三选一。（依据：需求内容第9条）
- 匿名进度与服务端合并时，采用更靠后的进度和更高完成状态。（依据：需求内容第14条）
- 章节排序值为大于等于1的整数，允许重复；客户端按排序值从大到小展示，同值按创建时间从新到旧展示。（依据：已审查原型 `course-detail.html`、`chapter-form.html`）
- 题目展示100字计数，选项和答案描述展示50字计数；其他字段及数量上限执行用户确认已补充的字段契约，具体值以权威 PRD 为准。（依据：已审查原型 `chapter-form.html`；用户确认，2026-07-18）
- 空专栏或零上架章节课程可在管理端保存但禁止上架；普通客户端仅展示满足发布条件的内容。（依据：用户确认，2026-07-18）
- 重复点击或并发领取使用同一奖励唯一键，不得重复发放。（依据：用户确认，2026-07-18）

## 十一、业务规则与数据契约

- 状态机流转规则：

| 当前状态 | 触发动作 | 前置条件 | 目标状态 | 失败提示 |
|---|---|---|---|---|
| 课程/章节下架 | 上架 | 资源存在 | 课程/章节上架，可作为正常内容展示 | 未提供 |
| 课程/章节上架 | 下架或删除 | 资源存在 | 不可作为正常可访问内容展示 | 未提供 |
| 章节未完播 | 播放达到完播判定进度 | 按最终生效播放设置执行 | 章节完播 | 未提供 |
| 章节完播、答题开启 | 提交答案 | 所有题目均答对 | 具备领奖资格 | 答错可继续答题，具体提示未提供 |
| 章节完播、答题关闭 | 进入领奖 | 奖励开关开启 | 具备领奖资格 | 未提供 |
| 奖励未领取 | 领取奖励 | 满足完播及答题条件 | 发放中或奖励已领取 | 重复请求返回原结果；失败保留可查询状态 |
| 奖励发放中 | 查询或重试 | 存在同一外部流水号 | 奖励已领取或发放失败 | 不创建新的领取记录 |
| 空专栏/零上架章节课程 | 执行上架 | 不满足最低可学习内容条件 | 保持下架 | 提示需先配置可上架课程或章节 |

- 数据契约约束：课程字段包括所属专栏、课程名称、课程简介、主讲人、课程封面、课程图文详情和上下架状态；章节包含所属课程、基本信息、播放设置和运营设置；播放策略包含进度条、手动暂停、快进、已完播快进、继续播放、随机暂停和完播判定进度；奖励类型为积分、优惠券、现金红包单选。（依据：需求内容第3、5、6、9条）章节排序值大于等于1且允许重复；题目100字，选项和答案描述50字。（依据：已审查原型 `course-detail.html`、`chapter-form.html`）其他字段执行用户确认的必填、格式、长度、数量、唯一性和错误提示契约；富文本按服务端白名单清洗；奖励唯一键为“租户 + 用户 + 章节 + 奖励配置版本”。（依据：用户确认，2026-07-18）

## 十三、BDD 验收标准 (Acceptance Criteria)

每个核心功能点至少覆盖 1 个成功路径、1 个校验失败和 1 个边界场景；未被原文定义的判断以推断编号标记，并在第十四章逐项追溯。

- **场景一：运营人员完成专栏、课程和章节配置**
  - **Given（前提）**：租户运营人员已进入运营 -> 内容运营 -> 专栏课。
  - **When（操作）**：新增专栏，在其下新增课程和章节，并保存表单。
  - **Then（结果）**：左侧展示专栏，右侧展示所选专栏的课程，课程详情展示基本信息和章节列表。
  - **验收方式**：页面结构与原型一致；课程详情不展示管理端【课程介绍】；章节所属课程固定且不可修改。（依据：需求内容第1、4、5条）

- **场景二：管理表单校验失败**
  - **Given（前提）**：运营人员正在新增或编辑课程/章节。
  - **When（操作）**：提交缺少必要业务信息或包含不合法内容的表单。
  - **Then（结果）**：系统阻止保存，保留已输入内容，并在对应字段展示可定位原因的提示。
  - **验收方式**：保存后列表和详情不存在本次无效变更；提示能指向具体字段和原因；字段限值与权威 PRD 一致。（依据：用户确认，2026-07-18）

- **场景三：管理对象能力边界**
  - **Given（前提）**：专栏下存在课程及章节。
  - **When（操作）**：运营人员查看课程和章节可用操作。
  - **Then（结果）**：课程不提供复制，章节提供复制；课程和章节均支持上下架，章节支持排序。
  - **验收方式**：课程操作区无复制入口；章节复制后产生可编辑章节；排序修改后自动保存并刷新。（依据：需求内容第2条；验收标准第5条）

- **场景四：章节使用租户默认播放策略**
  - **Given（前提）**：租户已配置专栏课默认播放策略，章节选择默认设置。
  - **When（操作）**：用户播放该章节视频。
  - **Then（结果）**：播放行为按同租户默认策略执行。
  - **验收方式**：进度条、暂停、快进、已完播快进、继续播放、随机暂停和完播判定分别与默认策略及现有课程系统一致。（依据：需求内容第6、12、13条）

- **场景五：默认播放策略缺失**
  - **Given（前提）**：租户未配置专栏课默认播放策略。
  - **When（操作）**：运营人员为章节选择默认设置。
  - **Then（结果）**：系统给出配置缺失提示或引导配置。
  - **验收方式**：页面出现可识别的缺失提示或可到达播放设置页的引导入口。（依据：验收标准第7条）

- **场景六：章节切换为自定义播放策略**
  - **Given（前提）**：租户存在默认策略，章节保存了不同的自定义策略。
  - **When（操作）**：用户播放该章节。
  - **Then（结果）**：章节自定义策略生效，不使用租户默认策略。
  - **验收方式**：逐项验证客户端行为与章节自定义值一致，且与现有课程系统相同配置的行为一致。（依据：需求内容第6、12、13条）

- **场景七：完播答题后领取奖励**
  - **Given（前提）**：用户已登录，章节已完播，答题和奖励开关开启，章节已配置多个题目和一种奖励。
  - **When（操作）**：用户答对所有题目后领取奖励。
  - **Then（结果）**：用户获得该章节配置的单一实际奖励。
  - **验收方式**：全部题目答对前不可领取；答对后显示可领取状态；领取结果的奖励类型与章节配置一致。（依据：需求内容第8、9、15条）

- **场景八：题目未全部答对**
  - **Given（前提）**：章节开启完播答题，用户已完播但至少一题答错。
  - **When（操作）**：用户提交答案并尝试领取奖励。
  - **Then（结果）**：用户不能获得奖励，可继续答题直到全部答对。
  - **验收方式**：奖励保持不可领取；再次作答入口可用；全部答对后领取资格才出现。（依据：验收标准第11条）

- **场景九：并发或重复领取奖励**
  - **Given（前提）**：用户已满足同一章节的领奖条件。
  - **When（操作）**：用户重复点击领取或多个请求并发到达。
  - **Then（结果）**：同一“租户 + 用户 + 章节 + 奖励配置版本”最多成功发放一次，后续请求返回原领取结果且不重复发奖。
  - **验收方式**：客户端最终仅展示一次成功领取结果；积分、优惠券或红包实际到账数量不因重复请求增加；超时状态可凭原外部流水号查询。（依据：用户确认，2026-07-18）

- **场景十：匿名用户播放并在登录后合并进度**
  - **Given（前提）**：未登录用户已在本地保存章节播放进度，服务端存在或不存在该用户历史进度。
  - **When（操作）**：用户登录并同步学习进度。
  - **Then（结果）**：使用更靠后的播放进度和更高完成状态合并到服务端。
  - **验收方式**：登录后进度不倒退，已完成状态不降级；未登录时提交答案按钮置灰。（依据：需求内容第14条；验收标准第12条）

- **场景十一：进度同步遇到网络失败**
  - **Given（前提）**：用户本地存在未同步进度。
  - **When（操作）**：登录同步过程中请求超时或中断。
  - **Then（结果）**：本地进度不丢失，客户端明确显示同步未完成，恢复网络后允许重新同步。
  - **验收方式**：失败后重新进入章节仍能恢复本地进度；未成功同步前服务端进度不被错误回退。（依据：用户确认，2026-07-18）

- **场景十二：进度与完成状态的合并边界**
  - **Given（前提）**：本地与服务端分别持有不同播放进度和完成状态。
  - **When（操作）**：用户登录触发合并。
  - **Then（结果）**：播放进度取更靠后值，完成状态取更高状态。
  - **验收方式**：覆盖“本地进度高但未完播”和“服务端进度低但已完播”组合，结果分别保留更靠后进度和已完播状态。（依据：需求内容第14条）

- **场景十三：从装修或分享入口访问专栏课**
  - **Given（前提）**：装修组件已配置专栏课，课程和章节正常可访问。
  - **When（操作）**：用户点击装修组件、推广分享链接或小程序分享卡片。
  - **Then（结果）**：用户进入对应专栏课详情或章节详情。
  - **验收方式**：装修组件显示课程名称和课程介绍；页面标题分别为“课程”或“章节”；章节列表标题为“章节目录”。（依据：需求内容第10、11条；验收标准第13、15条）

- **场景十四：分享资源已下架或删除**
  - **Given（前提）**：用户持有课程或章节的历史分享链接，但资源已经下架或删除。
  - **When（操作）**：用户打开该链接。
  - **Then（结果）**：资源不可作为正常可访问内容展示。
  - **验收方式**：不展示可播放的正常详情；具体失效页和提示文案未提供。（依据：验收标准第16条）

- **场景十五：富文本内容超过半屏**
  - **Given（前提）**：课程或章节配置了五段式图文混排详情，展示高度超过半屏。
  - **When（操作）**：用户首次进入详情页并点击【展开详情】。
  - **Then（结果）**：首次默认折叠并显示【展开详情】，点击后展示完整内容。
  - **验收方式**：课程和章节详情均执行相同折叠规则；未超过半屏时的按钮展示规则未提供。（依据：验收标准第14条）

- **场景十六：富文本包含不安全内容**
  - **Given（前提）**：运营人员在课程或章节富文本中录入脚本、危险链接或不受支持内容。
  - **When（操作）**：保存并在客户端渲染详情。
  - **Then（结果）**：系统按服务端白名单过滤或拒绝不安全内容，不执行脚本或危险跳转。
  - **验收方式**：管理端预览和客户端均不执行注入脚本；拒绝保存时提示具体不支持内容；未授权嵌入内容不可保存。（依据：用户确认，2026-07-18）

- **场景十七：空专栏或零章节课程**
  - **Given（前提）**：专栏没有课程，或课程没有已上架章节。
  - **When（操作）**：运营人员或客户端用户访问对应页面。
  - **Then（结果）**：管理端允许保存并展示可操作空状态，但禁止上架；普通客户端不展示不可学习内容。
  - **验收方式**：页面不报错、不出现无效播放入口；专栏至少包含一个已上架课程、课程至少包含一个已上架章节后才允许对普通用户展示。（依据：用户确认，2026-07-18）
